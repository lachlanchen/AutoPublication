> Автоматическая публикация видео с помощью инструментов ИИ.

## Обзор

`AutoPublication` предназначен для автоматизации рабочих процессов публикации видео с использованием инструментов, поддерживаемых ИИ.

На текущем этапе репозитория этот проект является каркасом с документацией и артефактами пайплайна генерации README, но исходный код приложения или исполняемая точка входа пока не зафиксированы.

Этот README служит канонической английской базой и структурирован так, чтобы поддерживать дальнейшее развитие реализации без потери текущего замысла проекта.

| Area | Current State |
|---|---|
| Implementation | Фаза каркаса (рабочее приложение ещё не добавлено) |
| Documentation | Канонический корневой README присутствует |
| Multilingual docs | Целевые языки определены в `i18n/` |
| Pipeline artifacts | Присутствуют в `.auto-readme-work/` |

## Возможности

- Настраиваемые интеграции для целевых платформ публикации.

## Заметки по разработке

- В репозитории пока нет исполняемого исходного кода.
- Генерация README, по-видимому, управляется пайплайном через файлы в `.auto-readme-work/`.
- Строка языковой навигации намеренно централизована в верхней части этого README для поддержки паритета между языками.

Предлагаемые следующие этапы реализации:
1. Добавить директорию исходного кода (например, `src/` или `autopublication/`).
2. Добавить манифест зависимостей (`pyproject.toml` или эквивалент).
3. Добавить исполняемую точку входа и минимальное покрытие тестами.
4. Добавить CI для проверки lint/test/docs.

## Устранение неполадок

### «Я не могу запустить проект»

Причина:
- Файлы рабочего приложения или точка входа пока не зафиксированы.

Решение:
- Рассматривайте репозиторий как каркас/документацию до добавления исходного кода.

### «Ссылки на языковые README есть, но файлы отсутствуют»

Причина:
- `i18n/` существует, но локализованные README-файлы в этом снимке ещё не сгенерированы.

Решение:
- Сгенерируйте/добавьте целевые файлы, перечисленные в `.auto-readme-work/20260228_230008/translation-plan.txt`.
